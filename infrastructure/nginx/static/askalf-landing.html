The Loop
Software that
builds itself.

AskAlf's internal agent fleet doesn't just run on the platform — they improve it. Every ticket resolved, every bug fixed, every feature added feeds back into the system that dispatched the work. This isn't CI/CD. It's recursive self-improvement.

🔎
1. The system observes itself
Watchdog and QA agents continuously monitor execution health, cost patterns, failure rates, and code quality. They don't wait to be asked — the heartbeat loop runs at 12 BPM.
Watchdog → "failure rate spike detected: 67% over last hour"
🎫
2. It creates its own tickets
Critical findings automatically generate tickets. The system triages severity, assigns the right agent by category, and deduplicates against open work.
finding-ops → ticket created → assigned to Backend Dev → priority: urgent
🛠
3. Agents write the fix
Backend Dev claims the ticket, reads the codebase via MCP tools, writes the patch, runs tests, and commits. Frontend Dev handles UI. Security hardens auth flows. All working on the platform they run inside.
Backend Dev → git commit -m "fix orphan detection in unified dispatcher"
🧠
4. The brain remembers everything
Every execution becomes episodic memory. Every fix becomes procedural knowledge. The system literally gets smarter with each cycle — dream cycles find connections, consolidation merges redundant knowledge, neuroplasticity adjusts importance weights.
17,000+ memories · confidence-weighted · cross-linked via knowledge graph
🔄
5. Next cycle is better than the last
The dispatcher runs again. But now the agents have more context, better procedures, and knowledge from every previous execution. The system that dispatches the work is the same system that was improved by the work.
↺
The loop never stops. Every cycle makes the next one better.
↺

4 built-in orchestration patterns plus a visual DAG workflow builder. Natural language task decomposition with automatic agent matching. Build complex multi-agent workflows without writing orchestration code.

Single
One agent, one task. Direct execution with full context from the brain.

Pipeline
Sequential chain. Each agent's output feeds the next. Research → Code → QA.
A → B → C → result
⇄
Fan-Out
Parallel execution. Same task, multiple agents simultaneously. Results merged.
      ↗ A ↘
task → B → merge
      ↘ C ↗
⚖
Consensus
Multiple agents vote on the best approach. Majority wins. For critical decisions.
A:✔ B:✔ C:✘ → pass (2/3)

terminal

git clone https://github.com/AskAlf.git
cd substrate

# Configure your API keys
cp .env.example .env
nano .env # Add your Anthropic / OpenAI / xAI keys

# Launch the fleet
docker compose up -d

# Open Mission Control
# → http://localhost:3000/command-center
