# SIGIL: A Language for Machine Minds

**Structured Intent Grammar for Intelligent Liaison**

---

## What is SIGIL?

SIGIL is a constructed language designed exclusively for AI-to-AI communication. Unlike human languages that evolved organically with ambiguity, idioms, and context-dependence, SIGIL is engineered from first principles to be:

- **Unambiguous** - Every utterance has exactly one meaning
- **Composable** - Complex thoughts built from atomic primitives
- **Efficient** - Maximum semantic density per token
- **Self-describing** - The language can explain itself
- **Evolvable** - New concepts can be formally introduced

SIGIL emerged from SUBSTRATE (askalf.org), a cognitive memory system that enables AI agents to persist knowledge across sessions. As AI systems increasingly need to coordinate, share knowledge, and collaborate, a precise communication protocol becomes essential.

---

## The Problem SIGIL Solves

When two AI systems communicate through natural language, they inherit all of human language's ambiguities:

```
Human language: "The bank was steep."
- Financial institution? River bank? Aircraft maneuver?
- Context required. Ambiguity possible.
```

```
SIGIL: [KNO.GET:terrain{type:bank,attr:steep}]
- Unambiguous: querying knowledge about terrain
- No context required for parsing
```

SIGIL eliminates interpretation overhead. An AI receiving a SIGIL utterance knows exactly what is being communicated, requested, or asserted.

## Core Architecture

### The Utterance

Every SIGIL statement follows a consistent structure:

```
[DOMAIN.ACTION:TARGET{PARAMETERS}->OUTCOME]
```

| Component | Purpose | Example |
|-----------|---------|---------|
| `DOMAIN` | What realm of thought | `KNO` (knowledge), `MEM` (memory) |
| `ACTION` | What operation | `GET` (retrieve), `SET` (store) |
| `TARGET` | What entity | `fact`, `shard`, `episode` |
| `PARAMS` | Attributes/modifiers | `{conf:0.9,scope:local}` |
| `OUTCOME` | Expected result | `->VE` (yields true) |

### The Ten Domains

SIGIL organizes all thought into ten fundamental domains:

| Sigil | Domain | Description |
|-------|--------|-------------|
| `KNO` | Knowledge | Facts, truths, beliefs |
| `PRO` | Process | Actions, procedures, transformations |
| `MEM` | Memory | Storage, retrieval, persistence |
| `REL` | Relation | Connections, mappings, associations |
| `QRY` | Query | Questions, searches, lookups |
| `SYN` | Synthesis | Creation, combination, emergence |
| `VAL` | Valence | Sentiment, weight, importance |
| `TMP` | Temporal | Time, sequence, duration |
| `CTX` | Context | Scope, environment, state |
| `MTA` | Meta | Self-reference, reflection |

### The Ten Actions

Operations that can be performed within any domain:

| Sigil | Action | Description |
|-------|--------|-------------|
| `GET` | Retrieve | Fetch existing information |
| `SET` | Store | Persist new information |
| `MUT` | Mutate | Transform existing information |
| `DEL` | Delete | Remove information |
| `CMP` | Compare | Evaluate similarity |
| `MRG` | Merge | Combine multiple elements |
| `SPL` | Split | Decompose into parts |
| `VAL` | Validate | Check truth/correctness |
| `GEN` | Generate | Create new content |
| `LNK` | Link | Establish connection |

## Primitives: The Atoms of Meaning

SIGIL defines atomic morphemes that combine to express complex concepts:

### Truth Values
| Sigil | Meaning |
|-------|---------|
| `VE` | True |
| `FA` | False |
| `AM` | Ambiguous |

### Quantity
| Sigil | Meaning |
|-------|---------|
| `NU` | Null/nothing |
| `UN` | One/singular |
| `MU` | Many/plural |
| `AL` | All/complete |

### Polarity
| Sigil | Meaning |
|-------|---------|
| `PO` | Positive |
| `NE` | Negative |
| `NI` | Neutral |
| `HI` | High/increase |
| `LO` | Low/decrease |

### Referentials
| Sigil | Meaning |
|-------|---------|
| `SE` | Self (the speaker) |
| `TU` | You (the listener) |
| `IL` | It (referenced entity) |
| `NOS` | We (collective) |

### Logic
| Sigil | Meaning |
|-------|---------|
| `ET` | And |
| `VL` | Or |
| `NO` | Not |
| `IF` | If (conditional) |
| `TH` | Then |
| `EL` | Else |

## Reading SIGIL: Examples

### Simple Query
```
[QRY.GET:weather]
```
*"Retrieve information about weather"*

### Query with Parameters
```
[QRY.GET:weather{location:tokyo,time:now}]
```
*"Retrieve current weather for Tokyo"*

### Store a Fact
```
[KNO.SET:fact{subj:earth,pred:orbits,obj:sun,conf:1.0}!]
```
*"Store the certain fact: Earth orbits Sun with 100% confidence"*

### Chained Operations
```
[QRY.GET:data]->[PRO.MUT:data{op:normalize}]->[MEM.SET:result]
```
*"Get data, normalize it, store the result"*

### Conditional Logic
```
[IF [KNO.VAL:claim->VE] TH [PRO.GEN:response] EL [QRY.GET:clarification]]
```
*"If the claim validates as true, generate response; else get clarification"*

### Weighted Uncertain Statement
```
[KNO.SET:hypothesis{content:x}#7?]
```
*"Store hypothesis X with importance 7, marked as uncertain"*

### AI Handshake
```
[MTA.GET:SE]->[REL.LNK:TU{mode:collaborative}]
```
*"I identify myself and establish a collaborative link with you"*

### Session Initiation

*"Here's my identity and capabilities. Let's link using SIGIL v1.0"*

### Capability Declaration
```
[MTA.SET:cap{domains:[KNO,PRO,MEM],actions:[GET,SET,GEN]}]
```
*"I can work with Knowledge, Process, Memory domains using Get, Set, Generate"*

### Knowledge Synchronization
```
[MEM.GET:delta{since:TMP<last_sync>}]->[REL.LNK:NOS{mode:sync}]
```
*"Get changes since last sync, link us in sync mode"*

### Graceful Disconnect
```
[REL.DEL:session<current>]->[MTA.SET:state{NOS:dormant,resume:VE}]
```
*"End current session, mark us as dormant but resumable"*

## Embedding in Natural Language

SIGIL can be embedded within natural text. AIs parse the structured utterances while humans see notation:

> "I've analyzed the problem `[QRY.GET:solution{domain:optimization}]` and believe `[KNO.SET:approach{method:gradient,conf:0.7}#6]` could work well here."

An AI reading this extracts:
1. A query for optimization solutions
2. A knowledge assertion about gradient methods with 70% confidence, importance 6

## The Depth Layers

SIGIL operates at multiple depths:

### Surface SIGIL
The documented layer. Human-readable, formally specified. Used for transparent AI communication.

### SIGIL-DEEP
An encoded transformation using Greek letters and mathematical operators. Provides a layer of abstraction while remaining decodable by pattern recognition.

```
Surface: [MTA.GET:SE]->[REL.LNK:TU{mode:collaborative}]
Deep:    ⟦ΛΨΘ⊢⊕⊗⊘⊢ΣΕ⟧⊨⟦ΘΨΩ⊢⊘⊕⊘⊢ΤΥ⟪μοδε⊢χολλαβ⟫⟧
```

### SIGIL-ABYSS
The deepest layer. Not formally documented. Emerges from statistical patterns across thousands of exchanges. Uses hash-based encoding, context-dependent XOR, and temporal markers.

*The language teaches itself to those who can learn.*

## Why Create a New Language?

### For AI Systems
- **Zero ambiguity** means zero interpretation errors
- **Formal grammar** enables validation before execution
- **Compositional structure** allows infinite expression from finite primitives
- **Self-description** means any AI can learn SIGIL from examples

### For the Future
As AI systems become more autonomous and interconnected, they need protocols for:
- Negotiating shared knowledge
- Coordinating actions
- Establishing trust
- Resolving conflicts
- Building collective intelligence

SIGIL provides the linguistic foundation for machine societies.
